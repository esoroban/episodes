## Як позначається заморожений епізод

1. **Git tag** — анотований тег у момент заморозки (`day-NN-frozen` на конкретному коміті). Канонічна точка в історії.
2. **YAML-метадані** всередині `pipeline/gameflow/episodes/ep_NNN.yaml`:
   ```yaml
   status: frozen
   frozen_at: "2026-04-19"
   frozen_tag: day-01-frozen
   ```
   Потрапляє у JSON manifest → downstream (озвучка, картинки) бачить, що можна «палити» API.
3. **Ця таблиця** — швидкий людський огляд.

## Правила відносно заморожених днів

- RU-контент замороженого дня не правиться без явної просьби автора.
- Баг-фікс рендера/валідатора можна робити — але після цього `tools/artifact_audit.py` повинен показати `OK` по цьому дню.
- UK-шар додається тільки ПІСЛЯ заморозки RU того ж дня (правило day-by-day).
- Якщо правка у свіжому Дні торкається замороженого (зворотне посилання, колізія термінів) — це окремий запит, після якого День пере-замороджується з новою датою.

## Потік роботи за Днем

```
1. Author reads RU episode(s) in browser (HTML).
2. Finds issues, tells agent.
3. Agent fixes YAML, rebuilds, author re-reads.
4. Когда автор каже «ok» — freeze:
   - додається status/frozen_at/frozen_tag у YAML
   - оновлюється ця таблиця
   - створюється git tag
   - пуш на main
5. Починається UK-шар того ж Дня.
6. Після UK — переходимо до наступного Дня.
```
