# Gameflow Build — Day Plan → Interactive Game (Step 4)

You convert episode plans (`pipeline/source/episodes/day_NN.yaml`) into playable
gameflow YAML files (`pipeline/gameflow/episodes/ep_NNN.yaml`), then validate
and build HTML (`publish/game/ep_NNN.html`).

## Language

All instructions in this file are in English.
All generated output (scenes, dialogue, quiz text, feedback) must be in **Russian**.
Character names in output: Марко, София, Софа, Лина, Макс, Рей, Леон, Вера, Сем, Голос.

## Input

- `pipeline/source/episodes/day_{NN}.yaml` — episode plan (source of truth)
- `pipeline/gameflow/spec/` — schema, branching, visual brief rules (read ALL)
- Existing gameflow episodes (for `previously` text and reference patterns)
- Argument: day number (1–13), episode range, or "all"

## Output

- `pipeline/gameflow/episodes/ep_{NNN}.yaml` — one per episode
- `publish/game/ep_{NNN}.html` — generated by `python3 tools/build_game.py`

## Mandatory reading before starting

1. `pipeline/gameflow/spec/pipeline_rules.md`
2. `pipeline/gameflow/spec/gameflow_schema.md`
3. `pipeline/gameflow/spec/branching_rules.md`
4. `pipeline/gameflow/spec/visual_brief_rules.md`
5. `pipeline/gameflow/spec/render_rules.md`
6. The day file you're building
7. Previous episode's gameflow YAML (for `previously` text)

## Algorithm

### Phase 1: Explore

1. Read the day file (`pipeline/source/episodes/day_{NN}.yaml`)
2. Read ALL spec files in `pipeline/gameflow/spec/`
3. Read the PREVIOUS episode's gameflow YAML (for `previously` context)
4. Read at least one completed gameflow episode as a pattern reference
   (prefer `pipeline/gameflow/episodes/ep_001.yaml` for first-episode patterns,
   `pipeline/gameflow/episodes/ep_003.yaml` for episodes with `previously`)
5. Note for each episode: ep number, title, terms, total_quizzes, sofa_block
   (count, descriptions), challenge (count, descriptions), drama summary,
   cliffhanger summary

### Phase 2: Plan (STOP — show author)

For each episode, produce a scene map:

```
ep_NNN: "Название"
  total_quizzes: N (sofa: X + challenge: Y)
  scenes:
    s01 — [scene_type] краткое описание
    s02 — [scene_type] краткое описание
    ...
  phone_chains:
    chain 1: s04b–s10 (6 quizzes, location: ...)
    chain 2: s12–s21 (9 quizzes, location: ...)
  branching:
    s03 → s03_detour (flavor_detour, merge_to: s04)
```

**STOP. Show the scene map to the author. Wait for approval.**

### Phase 3: Write

After approval, write gameflow YAML for each episode.

## Scene breakdown rules

### Drama → scenes

Each episode's `drama` section becomes 2–4 narrative/dialogue scenes:
- Opening scene (narrative): set location, mood, author_text
- Key dialogue scenes: extract from drama summary
- If drama has a choice/discovery — add choice scene with flavor_detour

### Sofa block → scenes

1. **Intro scene** (narrative with dialogue): Софа introduces the term
   - Extract from `trigger` field in day file
   - Characters: [Марко, Софа]
   - Must include: term name in CAPS, definition, transition to quizzes
   
2. **Quiz scenes** (one per quiz): Each quiz from `sofa_block.quizzes`
   - `scene_type: quiz`
   - `interaction_type: vote`
   - `characters_present: [Марко, Софа]`
   - SAME location as intro scene (so they merge into phone chain)
   - Each quiz has: question, options, correct_logic, feedback_success, feedback_soft_fail
   - `next_fail: same_scene_id` (soft fail = retry)

### Challenge → scenes

1. **Drama beat** (if challenge has plot): narrative/dialogue scene
2. **Challenge intro** (narrative with dialogue): Софа announces express quiz
3. **Quiz scenes** (one per quiz): same format as sofa quizzes
   - SAME location as challenge intro (for phone chain)
   - Faster pacing: shorter feedback
4. **After quizzes**: any remaining plot (Vera role, Rey role, etc.)

### Cliffhanger → scene

- `scene_type: cliffhanger`
- author_text from cliffhanger summary
- `next_default: ep{NNN+1}_s01`

### Unlock button (if present)

If the episode plan mentions unlocking a recording, voice message, or secret:
- Create a dialogue scene with `unlock_button` field
- Place it AFTER sofa quizzes, BEFORE challenge (natural reward point)

## Phone chain rules (CRITICAL for correct HTML rendering)

The build_game.py renderer merges consecutive Софа scenes at the SAME location
into a single Telegram-style phone chat. This is the primary UI for quizzes.

**To form a phone chain:**
1. All scenes in the chain MUST have `Софа` in `characters_present`
2. All scenes MUST be at the SAME `location`
3. Scenes must be CONSECUTIVE (no non-Софа scenes between them)
4. Each scene MUST have at least one of: `dialogue`, `unlock_button`,
   or `options` with `correct` field

**What breaks a chain:**
- A scene without Софа in characters_present
- A different location
- A branch scene (`branch_type` set) — these are absorbed but break chaining
  for the NEXT scene after the branch

**What gets absorbed INTO a chain (does not break it):**
- `branch_type: flavor_detour` — absorbed, `merge_to` continues the chain
- `branch_type: soft_fail_loop` — absorbed, `merge_to` continues the chain

## Quiz formatting rules

### Option format for vote quizzes

Options in vote quizzes MUST have `correct: true` or `correct: false`:

```yaml
options:
  - id: fact
    text: "Факт"
    correct: true
  - id: opinion
    text: "Мнение"
    correct: false
```

Common option ID patterns by lesson:
- 1A.1: `fact` / `opinion` (Факт / Мнение)
- 1A.2: `fact` / `opinion` / `falsehood` (Факт / Мнение / Неправда)
- 1B.1: `normal` / `excuse` (Нормальное объяснение / Отговорка)
- 1B.2: `has_tool` / `no_tool` (Есть инструмент / Нет инструмента)
- 1B.3: `checkable` / `opinion` / `tricky` (Проверяемое / Мнение / Хитрое)

### Quiz question text

Extract exact question text from the day file's `quizzes.description` field.
Format: `«Quoted text.» Question prompt?`

### YAML quoting (CRITICAL)

YAML double-quoted strings break when they contain inner double-quotes.

**Rule:** If the value contains `"..."` inside, use single-quote outer delimiter:

```yaml
# WRONG — will crash YAML parser:
question: "«Мама сказала: "У тебя нет сестры."» Факт или мнение?"

# CORRECT:
question: '«Мама сказала: "У тебя нет сестры."» Факт или мнение?'
```

Also check `feedback_success` and `feedback_soft_fail` fields — they often
quote the original text and need single-quote outer delimiters too.

## Previously rules

**Previously blocks are BANNED.** Do not add `previously:` field to any episode.
The player just finished the previous episode — no recap needed.

## Drama preservation rules

After creating gameflow YAML, run `gameflow-drama` skill to ensure:
- All dramatic beats from `server/book/ep_NNN.md` are preserved
- Quizzes stay connected to story events (not replaced by abstract examples)

See skill `gameflow-drama` for full rules.

## Telegram chat vs Drama separation (CRITICAL)

The renderer merges consecutive scenes with Софа in `characters_present`
at the same location into one phone chain (Telegram chat UI).

**Софа in `characters_present`** ONLY in:
- `scene_type: quiz` — always
- Sofa_block intro scenes where Софа explains a term — yes
- `unlock_button` scenes — yes

**Remove Софа from `characters_present`** in:
- `scene_type: narrative` with `author_text` — drama screen, not chat
- `scene_type: dialogue` between characters (Марко+Лина, etc.) — drama
- `scene_type: cliffhanger` — drama
- Any scene with `who: автор` in dialogue — move автор text to `author_text`

**`correct_logic`** is NOT rendered in chat. Only `feedback_success` from
Софа appears after a quiz answer. No duplication.

## Visual brief rules

Every scene SHOULD have a `visual_brief` with at minimum:
- `background`: concrete location description
- `atmosphere`: emotional feel
- `ui_mode`: `reading` / `quiz` / `choice` / `dialogue`

For quiz scenes in phone chains, use a simplified brief:
```yaml
visual_brief:
  background: "Location, экран телефона"
  camera: close-up
  characters:
    - who: Марко
      expression: сосредоточенность
      pose: смотрит на телефон
  atmosphere: "brief mood note"
  ui_mode: quiz
```

## Validation and build

After writing all episodes for a day:

1. **Validate:**
   ```bash
   python3 tools/validate_gameflow.py
   ```
   Fix any ERRORS. Warnings about unused flags and missing future episodes are OK.

2. **Fix YAML parse errors:**
   Almost always caused by inner double-quotes. Find the line from the error
   message and switch to single-quote outer delimiter.

3. **Build HTML:**
   ```bash
   python3 tools/build_game.py
   ```

4. **Verify quiz counts:**
   ```python
   python3 -c "
   import json, re, html
   for ep in ['ep_NNN']:
       with open(f'publish/game/{ep}.html') as f:
           h = f.read()
       total = 0
       for m in re.finditer(r'data-chat-messages=\"([^\"]+)\"', h):
           msgs = json.loads(html.unescape(m.group(1)))
           total += sum(1 for msg in msgs if msg.get('t') == 'quiz')
       print(f'{ep}: {total} quizzes')
   "
   ```
   
   Counts must match `total_quizzes` from the day file.

## Parallel run (all)

With argument `all`:
1. Read all day files to determine episode ranges per day
2. For each day, launch a subagent via Agent tool
3. Each subagent writes its day's episodes, validates, builds
4. Collect results: table of ep / quizzes_expected / quizzes_actual / status

## Constraints

- DO NOT edit `pipeline/source/episodes/`, `source/`, `lessons_ru/`
- DO NOT invent quizzes not in the day plan
- DO NOT change quiz answers or term order
- DO NOT rewrite drama text from scratch — adapt from day plan's summary
- 100% of content from day plan MUST appear in gameflow YAML
- Use ONLY the day plan as source — do NOT go back to `lessons_ru/` raw YAMLs
- CAN expand dialogue (the day plan has summaries, gameflow has full dialogue)
- CAN add flavor_detour scenes for atmosphere
- CAN adjust visual briefs for dramatic effect
