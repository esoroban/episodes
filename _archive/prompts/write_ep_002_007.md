# Промпт: написать эпизоды 2–7 книги «Сила Слова»

## Контекст

Проект «Сила Слова» — интерактивная образовательная драма для детей 8–12 лет.
25 уроков критического мышления вплетены в фэнтези-триллер.

Pipeline построен. Шаги 0 (брифы), 0-QA, 1 (grid) завершены. Эпизод 1 написан
и утверждён автором. Нужно написать эпизоды 2–7 в том же формате.

## Что прочитать для восстановления контекста

Читай в этом порядке:

1. `CLAUDE.md` — правила проекта, персонажи, запреты
2. `context_state.md` — текущее состояние
3. `decision_log.md` — все решения автора (ОБЯЗАТЕЛЬНО — там баги и правила)
4. `pipeline/pipeline.md` — шаги пайплайна
5. `pipeline/style_profile.yaml` — стиль прозы (school-mystery + detective clue chain)
6. `pipeline/grid.yaml` — грубая сетка (какие блоки в каких эпизодах, термины, votes)

## Эталон — утверждённый эпизод 1

Прочитай внимательно:
- `book/ep_001.md` — ГОТОВЫЙ ЭПИЗОД. Это эталон формата и качества.
- `pipeline/review/ep_001_review.html` — HTML-вычитка (слева урок, справа книга)

## Что нужно сделать

Написать эпизоды 2–7 (День 1, эпизоды 2-4 + День 2, эпизоды 5-8).
Для каждого эпизода:

### 1. Прочитать входные данные

- **Source** (`source/СИЛА_СЛОВА_40_ЭПИЗОДОВ.md`): прочитать ТОЛЬКО source-эпизоды
  для нужного дня (из grid.yaml → source_episodes). Source — ФУНДАМЕНТ драмы.
  НЕ переписывать. Использовать как основу.
- **Brief** (`pipeline/briefs/brief_{ID}.yaml`): прочитать брифы уроков дня.
  Оттуда: квизы (practice_summary), теория (key_material, summary), определения.
- **Grid** (`pipeline/grid.yaml`): порядок терминов, votes, story_beat для каждого эпизода.
- **YAML урока** (`lessons_ru/lesson_{ID}.yaml`): оригинальные тексты квизов (ru-ключи).

### 2. Написать эпизод (`book/ep_NNN.md`)

Формат — как ep_001.md:
- **[ДРАМА]** — из source. Расширить, добавить деталей, диалогов, gut feelings.
  НЕ СОКРАЩАТЬ source. Добавлять, не убирать. Драма БОЛЬШЕ урока, не меньше.
- **[СОФА-БЛОК]** — Софа вводит термин через ситуацию из драмы. Квизы из брифа,
  но маппинг на сюжет (generic → story). Определения из урока. Правило Софы.
- **[ИСПЫТАНИЕ]** — драматический вызов + квизы в действии. Роль Веры/Рея/Голоса
  если есть. Изъян Марко (соглашатель).
- **[КЛИФФХЭНГЕР]** — из source. Твист. Must-watch-next.

### 3. Проверки (КРИТИЧЕСКИЕ, перед показом)

**Порядок терминов:**
- Построить кумулятивный terms_available для каждого эпизода
- Квизы — ТОЛЬКО из доступных терминов
- Если эпизод бинарный (факт/мнение) — квизы бинарные, без «неправды»

**Корректность контента:**
- Каждый ответ однозначно правильный по определениям урока
- Субъективные оценки = ВСЕГДА мнение (нет скучномера, красивомера...)
- «Ты переутомился» = мнение, НЕ факт

**Утечки из будущих уроков:**
- «Если не съешь суп — заболеешь» — это манипуляция страхом (День 8), НЕ использовать в Дне 1
- Если пример требует инструмента из будущего урока — НЕ использовать

**Спойлеры:**
- Лина — не называть агентом до Дня 9
- Вера — не называть манипулятором до Дня 3+
- Стирания — не упоминать до Дня 3

### 4. HTML-вычитка (`pipeline/review/ep_001_review.html`)

Добавить кнопки «Эпизод 2», «Эпизод 3» и т.д. в навигацию.
Для каждого нового эпизода — секция с двумя колонками:
- **Слева** — ЧИСТЫЙ урок из YAML. Прямая речь учителя. Квизы. Определения.
  Никакого сюжета. Только урок.
- **Справа** — КНИГА. Полная драма. Квизы вплетены в историю.
  НЕ краткий пересказ, а полный текст.

## Правила

- Имена КИРИЛЛИЦЕЙ: Марко, София, Софа, Лина, Макс, Рей, Леон, Вера, Сем, Голос
- Source — read-only, НЕ редактировать
- Из YAML — только ru-ключи
- Стиль: close Marko focalization, short sentences, concrete, no essay language
- Каждая сцена: вопрос → проверка → противоречие → улика → последствие
