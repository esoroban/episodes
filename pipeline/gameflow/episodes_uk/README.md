# UK Overlay — правила перевода

## Архитектура

UK-версия игры **не самостоятельный файл**, а оверлей поверх RU source:

```
pipeline/gameflow/episodes/ep_NNN.yaml      (RU source — структура + текст)
pipeline/gameflow/episodes_uk/ep_NNN.yaml   (UK overlay — ТОЛЬКО текст)
          │
          ▼  python3 tools/build_game.py --lang uk
          │
server/game/uk/ep_NNN.html + ep_NNN.json    (сгенерированная UK-игра)
```

`build_game.py` при `--lang uk` мёрджит переводы из overlay в RU-структуру.
Всё, что не переопределено в overlay, берётся из RU.

**Следствие:** править `server/game/uk/*` вручную бессмысленно — следующий
`build_game.py --lang uk` перезапишет. Все изменения делаем в overlay YAML.

## Что переводим (текст)

Только эти поля overlay'а переносятся в UK-сборку — см. `_TEXT_FIELDS` и
`merge_uk_overlay()` в [tools/build_game.py](../../../tools/build_game.py):

### На уровне эпизода
- `episode_title` — название эпизода (в кавычках-ёлочках)
- `terms_introduced` — список терминов (перевод на украинский)

### На уровне сцены
- `author_text` — авторский текст до диалога
- `author_text_after` — авторский текст после диалога
- `dialogue[].line` — реплики (по индексу)
- `options[].text` — варианты выбора (matched by `id`, не по порядку)
- `question` — вопрос квиза
- `correct_logic` — логика правильного ответа (хинт)
- `feedback_success` — фидбек при правильном ответе
- `feedback_soft_fail` — мягкий фидбек при ошибке
- `mood` — настроение сцены
- `interactions[].{question,correct_logic,feedback_*,options[].text}` — множественные квизы в одной сцене
- `followup_interaction.*` — доп. квиз после сцены
- `unlock_button.text` — текст кнопки разблокировки
- `unlock_button.reveals.{line,who}` — раскрываемая реплика

## Что НЕ переводим

Эти поля **всегда берутся из RU** — overlay их игнорирует:

| Поле | Почему |
|------|--------|
| `scene_id` | Идентификатор, код |
| `kind`, `is_chat`, `branch_type`, `ui_mode` | Тех. флаги |
| `location`, `time`, `characters_present` | Технические метаданные — скрыты в UK HTML |
| `visual_brief.*` (background, camera, atmosphere, focus_object, characters, props) | Брифинг для художников — скрыт в UK HTML |
| `set_flags`, `require_flags`, `nav.*` | Игровая механика |
| `options[].id` (ключ) | Идентификатор варианта |
| `dialogue[].who` | Имя в YAML остаётся на рабочем языке. В UK-билд имена подставляются через `_UK_WHO_DISPLAY` в [tools/build_game.py](../../../tools/build_game.py) (Марко→Марко, Софа→Софа, Подпольщица→Підпільниця, горожане подполья→мешканці підпілля и т.д.). Если появляется новый персонаж — добавить его в эту мапу. |
| `source_ref` | Источник |

**`visual_brief` и `scene-meta` скрыты в UK HTML через CSS** (`body.lang-uk ... { display: none }`),
поэтому даже если там русский текст — игрок его не увидит. Художники работают с RU YAML напрямую.

## Имена персонажей (глоссарий)

В YAML поле `who` — на рабочем языке (русский Cyrillic). В переводимом тексте (`line`, `author_text`) — украинские формы:

| RU (исх.) | UK (в тексте) |
|-----------|---------------|
| Марко | Марко |
| София | Софія |
| Софа | Софа |
| Лина | Ліна |
| Макс | Макс |
| Рей | Рей |
| Леон | Леон |
| Вера / Вера Андреевна | Віра / Віра Андріївна |
| Сем | Сем |
| Голос | Голос |

Применяем украинские падежные формы: Марку (дат.), Софію (вин.), Ліни (род.),
Ліно (кличний), и т.д.

Второстепенные персонажи уроков (Дмитрик, Вітя, Маша, Микола, Олена, Данило) —
транслитерируем в украинскую форму.

## Процесс: как перевести новый эпизод

1. **Скопируй шаблон** из соседнего overlay (например, `ep_001.yaml`) в `ep_NNN.yaml`.
2. **Открой RU source** `pipeline/gameflow/episodes/ep_NNN.yaml` рядом.
3. **Перенеси только переводимые поля** (см. список выше), НЕ копируй `visual_brief`, `location`, `time`, `characters_present`, `set_flags`, `nav.*`.
4. **Поле `who`** в `dialogue[]` оставляй на русском (рабочий язык).
5. **Сохраняй порядок** сцен и реплик — мёрдж идёт по `scene_id` и по индексу реплики.
6. **Варианты выбора** мёрджатся по `id` — следи, чтобы id в overlay совпадали с RU.
7. **Собери и проверь:**
   ```bash
   python3 tools/build_game.py --lang uk ep_NNN
   ```
8. **Глазами проверь** [server/game/uk/ep_NNN.html](../../../server/game/uk/):
   - нет русских фраз в тексте, диалогах, квизах
   - имена в украинских падежах
   - термины переведены по глоссарию

## Команды

```bash
# Собрать всё
python3 tools/build_game.py                  # RU
python3 tools/build_game.py --lang uk        # UK

# Собрать один эпизод
python3 tools/build_game.py ep_005
python3 tools/build_game.py --lang uk ep_005

# Валидатор (ловит битые ссылки, неиспользованные флаги)
python3 tools/validate_gameflow.py
```

`build_game.py` чистит выходную директорию перед билдом (`Cleaned server/game/ru/`
или `.../uk/`) — после редактирования **всегда пересобирай полностью**, иначе
удалённые эпизоды пропадут из выхода.

## Что делать, если перевод не влезает в схему overlay

Если нужно перевести поле, которое сейчас **не** в `_TEXT_FIELDS` (например,
какое-то новое поле, появившееся в YAML-схеме):

1. Добавь ключ в `_TEXT_FIELDS` в [tools/build_game.py](../../../tools/build_game.py) или расширь `merge_uk_overlay()`.
2. Обнови этот README — добавь поле в раздел «Что переводим».
3. Перегенерируй UK-билд.

Не пытайся править `server/game/uk/*` напрямую — перезапишется.
