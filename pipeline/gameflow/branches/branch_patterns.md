# Branch Patterns — Каталог паттернов ветвлений

## Принцип

Главная линия — коридор. Ветвления — петли, которые ВСЕГДА возвращаются
в ту же точку main line. Глубина: 1–3 сцены. Ветвления не содержат quiz.
Ветвления содержат: атмосферу, лор, эмоции, улики, диалоги.

## Паттерны

### A. Peek (1 сцена)
Текущий стандарт. Быстрый взгляд — и обратно.
```
main: ──[s03]──────────[s04]──
              │          ▲
              └──[s03d]──┘
```
Пример: ep_001 — полистать дневник Софии.

### B. Investigation Loop (2 сцены)
Игрок исследует что-то. Сцена 1 — обнаружение. Сцена 2 — реакция/вывод.
```
main: ──[s03]──────────────────[s05]──
              │                  ▲
              └──[s03d1]→[s03d2]─┘
```
Пример: осмотреть комнату Леона → найти записку → прочитать записку → merge.

### C. Conversation Fork (2 сцены на каждый путь)
Два варианта разговора. Оба дают разную информацию, оба ведут в одну точку.
```
main: ──[s03]──────────────────────[s06]──
              │                      ▲
              ├──[s03a1]→[s03a2]─────┤
              └──[s03b1]→[s03b2]─────┘
```
Пример: поговорить с Линой о прошлом / поговорить о настоящем.

### D. Emotional Spiral (2 сцены)
Выбор эмоциональной реакции. Разные сцены — одна рефлексия в конце.
```
main: ──[s03]──────────────────[s05]──
              │                  ▲
              └──[s03d1]→[s03d2]─┘
```
Пример: крикнуть в ответ → пожалеть → merge. Разница с B: здесь внутренний
мир, не внешнее исследование.

### E. Observation Chain (3 сцены)
Цепочка наблюдений. Каждая сцена — ещё одна деталь. Нарастание.
```
main: ──[s03]────────────────────────────[s06]──
              │                            ▲
              └──[s03d1]→[s03d2]→[s03d3]───┘
```
Пример: ep_004 расширенный — тень отстаёт → отражение не совпадает →
шёпот стен → merge. Три аномалии = паттерн.

### F. Try-and-Learn (2 сцены)
Игрок пробует действие. Оно не срабатывает (или срабатывает неожиданно).
Урок из попытки.
```
main: ──[s03]──────────────────[s05]──
              │                  ▲
              └──[s03d1]→[s03d2]─┘
```
Пример: попробовать открыть дверь → дверь не поддаётся, но на ручке
остаётся след → merge.

### G. Parallel Discovery (2+2 сцены, два пути)
Два способа узнать одно и то же. Игрок выбирает, КАК узнать.
```
main: ──[s03]──────────────────────[s06]──
              │                      ▲
              ├──[s03a1]→[s03a2]─────┤
              └──[s03b1]→[s03b2]─────┘
```
Пример: спросить Софу / осмотреть самому. Оба пути приводят к одному факту,
но разный опыт.

## Правила для всех паттернов

1. Ветвления НЕ содержат quiz (quiz — только в phone chain)
2. Каждая ветка имеет `merge_to` — возврат в main line
3. Максимальная глубина — 3 сцены от точки ветвления
4. Ветвления в drama-секциях (до phone chain, между chains, после chains)
5. Сцены ветвлений должны раскрывать: персонажа, мир, улику или эмоцию
6. Не повторять информацию из main line — ветвление = бонус, не дубль
