# Branching Rules — Правила ветвлений

## Принцип: коридор с карманами

Игра — линейный коридор с короткими ответвлениями. Не дерево. Все ветки возвращаются в main line в пределах 1-3 сцен.

```
main:  ──[s01]──[s02]──[s03]──────────[s05]──[s06]──
                  │                      ▲
                  └──[s02_fail]──────────┘
                       (подсказка)
```

## Разрешённые типы ветвлений

### 1. soft_fail_loop
**Когда:** игрок ответил неправильно на quiz.
**Что происходит:** короткая сцена с подсказкой, возврат к тому же вопросу или следующему.
**Глубина:** 1 сцена.
**Merge:** автоматический, в `next_default` основной сцены или в `merge_to`.

```yaml
# Основная сцена
- scene_id: ep001_s04
  next_default: ep001_s05
  next_fail: ep001_s04_fail

# Тупик
- scene_id: ep001_s04_fail
  branch_type: soft_fail_loop
  merge_to: ep001_s04  # возврат к вопросу
```

### 2. flavor_detour
**Когда:** игрок выбирает «исследовать» / «поговорить ещё» / «осмотреть предмет».
**Что происходит:** 1-2 дополнительные сцены с атмосферой, лором или эмоцией.
**Глубина:** 1-2 сцены.
**Merge:** в следующий узел main line.

```yaml
- scene_id: ep001_s03
  interaction_type: choice
  options:
    - id: look_diary
      text: "Полистать дневник"
      next: ep001_s03_detour
    - id: go_on
      text: "Идти в школу"
      next: ep001_s04
  next_default: ep001_s04

- scene_id: ep001_s03_detour
  branch_type: flavor_detour
  merge_to: ep001_s04  # возвращаемся в main line
```

### 3. gated_response
**Когда:** есть флаг, меняющий реакцию NPC.
**Что происходит:** разный текст, но тот же следующий узел.
**Глубина:** 0 (разные тексты в одной сцене).
**Merge:** не нужен — сцена одна.

```yaml
- scene_id: ep001_s07
  dialogue:
    - who: Софа
      line: "Ты нашёл дневник. Это факт."
      require_flags: [marko_found_diary]
    - who: Софа
      line: "Ищи доказательства."
      require_flags_absent: [marko_found_diary]
  next_default: ep001_s08
```

### 4. cosmetic_branch
**Когда:** выбор влияет на тон, но не на структуру.
**Что происходит:** ставится флаг, меняющий реплики в будущих сценах.
**Глубина:** 0 сцен (только флаг).
**Merge:** не нужен.

```yaml
- scene_id: ep001_s06
  interaction_type: choice
  options:
    - id: trust
      text: "Верю тебе, Лина"
      set_flags: [trust_lina]
    - id: doubt
      text: "Мне нужно подумать"
  next_default: ep001_s07  # одинаковый для обоих
```

## Запрещённые ветвления

| Запрещено | Почему |
|-----------|--------|
| Параллельные полноценные сюжетные линии | Нет бюджета на дублирование контента |
| Деревья глубиной > 3 сцен | Теряется контроль, растёт объём |
| Необратимые развилки без merge | Каждая ветка должна вернуться |
| Уникальные сцены на каждый вариант ответа | Слишком дорого; используй gated_response |
| Ветвления, требующие «помнить» больше 5 флагов | Слишком сложная логика |

## Правила merge

1. **Каждая ветка имеет `merge_to`** — ID сцены main line, куда она возвращается.
2. **Merge происходит максимум через 3 сцены** от точки ветвления.
3. **Soft fail всегда возвращает** к тому же вопросу или к следующему.
4. **Flavor detour никогда не содержит quiz** — это чистая атмосфера.

## Допустимые флаги

Флаги — простые boolean. Примеры:

| Флаг | Когда ставится | Статус |
|------|----------------|--------|
| `sofa_activated` | ep_001: Софа включилась | **ИСПОЛЬЗУЕТСЯ** — `require_flags` в ep_002, ep_003, ep_004 |
| `marko_found_diary` | ep_001: Марко нашёл дневник | Резерв — для будущих gated_response |
| `marko_saw_drawing` | ep_003: Марко видел рисунок Лины | Резерв — для будущих gated_response |
| `door_confirmed` | ep_004: дверь подтверждена тремя проверками | Резерв — для будущих эпизодов |
| `spotted_reflection` | ep_004: Марко заметил аномалию отражения Лины | Резерв — для будущих эпизодов |
| `trust_lina` | (не используется пока) | Резерв |
| `trust_vera` | (не используется пока) | Резерв |
| `spotted_contradiction` | (не используется пока) | Резерв |
| `soft_fail_count` | Счётчик ошибок (int, исключение из boolean) | Резерв |

**Максимум 10 активных флагов на эпизод.** Не RPG-система.

### Неиспользованные флаги

Флаги со статусом «Резерв» ставятся (`set_flags`), но пока не требуются (`require_flags`). Это нормально на текущем этапе — они заработают когда:
- появятся gated_response сцены с разным текстом в зависимости от флага
- будет больше эпизодов, где выбор из ep_001-004 влияет на реплики

Валидатор (`tools/validate_gameflow.py`) отслеживает неиспользованные флаги как warnings. Когда все Day 1-13 будут готовы, оставшиеся «Резерв» флаги нужно либо задействовать, либо удалить.

Для каждого эпизода рекомендуется ASCII-карта потока в комментарии в начале файла:

```
# Flow: s01 → s02 → s03 ──→ s04 → s05 → s06 → s07
#                  │    ↗        ↑
#                  └→ s03d ──┘   │
#                       s04f ────┘
```
