## 3. Слой image_prompts.yaml

Для каждой drama-сцены (экран с иллюстрацией поверх) — промпт
для image-модели.

Что уже есть, что поможет:
- `visual_brief` в каждой сцене — background, atmosphere, camera,
  characters с expression/pose, props, focus_object.
- Это именно то сырьё, из которого собирается промпт.

Что нужно добавить:
- Style-pack: единый визуальный стиль всего курса (школа-детектив,
  close focalization, палитра).
- Character-pack: канонические описания каждого героя (Марко в сером
  свитере, Софа = треснутый экран, …). Чтобы один и тот же персонаж
  выглядел одинаково на 50 эпизодах.
- Режиссёрские директивы на уровне сцены (ракурс, свет, настроение).

Итог: промпт-сцена = style-pack + character-pack + visual_brief + mood.

**Скрипт `tools/build_image_prompts.py`** — сгенерировать из gameflow.

## 4. Слой voice_prompts.yaml

Современные TTS-модели (включая ту, что вышла на этой неделе)
принимают не только текст, но и режиссёрские директивы:
«шёпотом», «с придыханием», «пауза 0.5 сек», «улыбкой в голосе».

Для каждой реплики нужно два поля:
- `text` — что сказать (субтитр, уже есть в gameflow).
- `direction` — как сказать (тон, эмоция, темп).

Структура:

```yaml
scene_id: ep001_s04
voice_tracks:
  - id: ep001_s04_author_01
    speaker: автор
    text: "Шаги за стеной. Ближе. Ещё ближе."
    direction:
      emotion: тревога-сдержанная
      pace: замедленный
      volume: шёпот-на-грани
      pause_after_ms: 400
  - id: ep001_s04_marko_01
    speaker: Марко
    text: "София?"
    direction:
      emotion: надежда-страх
      volume: полушёпот
```

Что уже есть:
- Реплики в `dialogue` / `dialogue_after` — это text.
- `mood` сцены — сырой сигнал для direction.
- `voice_message.line + duration` — аналог уже есть в unlock_button.

Что нужно добавить:
- Voice-pack: профили голосов (тембр, возраст, акцент) на каждого персонажа.
- Сборщик: из gameflow + voice-pack → voice_prompts.yaml.

**Скрипт `tools/build_voice_prompts.py`** — один source, один output.

## 5. Что это меняет в проектировании СЕЙЧАС

Чтобы будущий переезд был дешёвым, сейчас делаем так:

- **Каждая реплика в `dialogue` имеет чёткого `who`.** Уже есть.
- **`mood` сцены заполняем всегда**, даже если сейчас не рендерится.
  Это сырьё для voice direction.
- **`visual_brief` обязателен для drama-сцен** (пока опционален).
  Это сырьё для image prompts.
- **Не хардкодить стилистику в рендере.** Стили — в packs, не в `build_game.py`.
- **Не смешивать текст и направление голоса.** Не писать «Марко *шёпотом*:
  — София?» — направление должно быть отдельным полем, не в строке реплики.

То есть: уже сейчас воспитываем дисциплину «text отдельно, direction отдельно»,
даже когда direction ещё не используется.

## 6. Legacy pipeline (архив для других курсов)

Старый путь:
```
lesson-brief → lesson-map → story-grid → episode-plan → episode-map
```

Использовался для генерации 50 планов эпизодов SylaSlovaDramma.
Все 50 уже прошли этот путь — результат в `pipeline/source/episodes/day_*.yaml`.

**Статус:** legacy для этого курса. Активный входной слой сейчас —
`pipeline/source/episodes/` (готовые планы).

**Почему сохраняем:** при запуске нового курса (другая книга + другие уроки)
весь этот путь заработает заново. Скиллы и их спецификации остаются в
`.claude/skills/`.

**Следствие:** пайплайн-диаграмма в COMPLETE_RULES.md должна показывать
**два режима**:
- Новый курс с нуля — полный legacy path.
- Активная работа с SylaSlovaDramma — начинаем с gameflow-build.
