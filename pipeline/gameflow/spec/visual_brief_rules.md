# Visual Brief Rules — Правила визуальных брифов

## Назначение

Visual brief — не финальная картинка, а **техническое задание** на один «кадр» игры. Из него потом генерируется арт, фон, спрайт или UI-макет.

## Где находится

Visual brief — **часть сцены** в gameflow YAML (поле `visual_brief`). Отдельные файлы в `pipeline/gameflow/visuals/` создаются только для batch-экспорта или специальных правил.

## Обязательные поля

| Поле | Тип | Описание | Пример |
|------|-----|----------|--------|
| `background` | string | Описание фона | `Комната Марко, утренний свет` |
| `atmosphere` | string | Общее настроение кадра | `тихо, тревожно` |
| `ui_mode` | enum | Режим интерфейса | `reading`, `quiz`, `choice`, `dialogue` |

## Опциональные поля

| Поле | Тип | Описание | Пример |
|------|-----|----------|--------|
| `camera` | string | Ракурс камеры | `close-up`, `wide`, `over-shoulder`, `pov` |
| `characters` | list[object] | Персонажи в кадре | см. ниже |
| `props` | list[string] | Важные предметы | `[дневник, телефон с трещиной]` |
| `focus_object` | string | На чём фокус | `треснутый экран телефона` |

## Структура персонажа в visual_brief

```yaml
characters:
  - who: Марко
    expression: растерянность    # эмоция на лице
    pose: сидит на кровати       # поза / действие
    position: center             # left, center, right (опционально)
```

## Допустимые ui_mode

| Режим | Когда |
|-------|-------|
| `reading` | Авторский текст, нарратив |
| `dialogue` | Диалог (текстовый бокс внизу) |
| `quiz` | Вопрос с вариантами ответов |
| `choice` | Сюжетный выбор игрока |
| `feedback` | Реакция на ответ |
| `transition` | Смена локации / перебивка |

## Правила написания

1. **Пиши конкретно, не абстрактно.** Не «Марко грустит», а «Марко сидит на кровати, смотрит на пустую кровать напротив, руки на коленях».
2. **Один кадр — один фокус.** Не описывай всё, что теоретически есть в комнате. Только то, что важно для этого момента.
3. **Background меняется только при смене локации.** Если три сцены подряд в одной комнате — background одинаковый.
4. **Camera по умолчанию — `wide`.** Меняй только когда есть причина.
5. **Props — только сюжетно важные.** Не перечисляй мебель. Только то, что двигает историю.

## Batch-экспорт

Для генерации арта можно экспортировать все visual_brief из эпизода в отдельный файл:

```
pipeline/gameflow/visuals/ep_001_visuals.yaml
```

Это делается скриптом, не вручную. Формат:

```yaml
episode: 1
scenes:
  - scene_id: ep001_s01
    background: "Комната Марко, утренний свет"
    camera: close-up
    characters: [...]
    props: [...]
    atmosphere: "тихо, тревожно"
```

## Чего НЕ должно быть в visual brief

- Финальных размеров и разрешений (это задача рендера)
- Стилистических указаний (аниме, реализм) — это решается на уровне art pipeline
- Цветовых кодов — только словесные описания
- Анимаций — только статичные кадры
