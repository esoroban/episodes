# QA findings (live build)

Диагностические находки из проверки задеплоенной игры.

## F1. UK visual_brief leakage (RU данные на UK-страницах)

**Симптом:** На UK-странице ep_004/s05 пользователь видит:
- `Марко:: Марко: — Ліно, можна я перевірю?`
- `Марко:: Лина: — Давай.`

**Причина:** UK YAML (`pipeline/gameflow/episodes_uk/ep_NNN.yaml`) — текстовый оверлей. Поле `visual_brief` в нём отсутствует. Рендерер `build_game.py` подтягивает `visual_brief` из RU, который содержит русские лейблы персонажей (`Марко:`, `Лина:`). При показе:
1. Блок visual-brief с лейблом `Марко: решимость, готовится...`
2. Сразу за ним dialogue-блок `Марко: — Ліно...`

→ читатель видит «двойного Марко» и русскую «Лина:» рядом с украинской «Ліна:».

**Скоуп:** проблема не локальна для s05 — она системная для всех UK-эпизодов, где есть visual_brief с именами персонажей.

**Варианты фикса (на обсуждение):**
- (а) Перевести visual_brief в UK YAML (большая работа, оверлей должен покрывать ещё одно поле).
- (б) В рендерере `build_game.py` локализовать только лейблы (`Лина` → `Ліна` и т.д.) при сборке UK.
- (в) Скрыть визуальные подписи имён в visual-brief на странице (они не нужны игроку — это бриф для художника).

**Рекомендация:** (в) — visual_brief нужен художнику, не игроку. Скрыть в рендере UK (и, возможно, RU).

## F2. ep_007 «обрыв» после s03 — две причины: нет картинки + звук

**Симптом:** После реплики Софы «Число впало. 47 → 46 → 44...» в задеплоенной UK-версии ничего не воспроизводится — ни голос, ни визуал.

**Проверка данных:**
- UK YAML: 17 сцен, до `ep007_s10_sofa` — целый.
- Собранный JSON `server/game/uk/ep_007.json`: 1207 строк, все 17 сцен присутствуют, в s03 все 6 реплик есть.
- HTML тоже содержит все сцены.

**Проверка ассетов (по `server/v2/uk/ep_007/ep_007.html`):**
Картинки на R2 есть для: s01, s02, s07, s07_choice, s07c_use, s07c_save, s09, s10.
**Картинок НЕТ для s03 и s05** — оба narrative-сцены, должны иметь визуал.

**Вывод:** обрыв = совпадение двух дыр на одной сцене:
1. **Нет картинки для s03** → визуально страница «застревает» на старой картинке (s02) или пустоте.
2. **Сломанный/отсутствующий audio-трек** для одной из реплик s03 → плеер фризит ожидание озвучки, переход к следующей сцене не происходит.

**Действия:**
1. Сгенерировать картинки для ep_007 s03 и s05 (записаны в needed_images.md).
2. Проверить combined audio для ep_007 в R2 — сверить количество треков с количеством реплик YAML.
3. Если track сломан — пересоздать combined audio для ep_007 и повторно залить в R2.
